# Task 10 - PostgreSQL App Schema

Branch: `task/010-postgresql-app-schema`

## Scope

This task adds the app-owned PostgreSQL schema for PreOdds web and mobile clients.

It covers users, authentication identities, user preferences, mobile/web devices, favorites, featured fixtures, tips, coupons, notifications, notification deliveries, and contact messages. It does not add legacy historical migration, SportMonks sync workers, API clients, API controllers, or UI changes.

Task 9 is intentionally skipped because historical data transfer will not be performed. New data will be populated from SportMonks API and app runtime flows.

## Design Decisions

- App-owned primary keys are UUIDs generated by PostgreSQL with `gen_random_uuid()`.
- Users are not tied to the old `prd_user` numeric ID model.
- Email and username uniqueness are handled with normalized unique indexes using `lower(...)`.
- Authentication identities are separated from `app.users` so email, Google, Apple, Facebook, Twitter, and Microsoft login methods can coexist.
- `app.user_devices` supports web, iOS, and Android clients and stores push-token metadata without forcing every device to belong to a logged-in user.
- Favorites use explicit foreign keys for teams, leagues, and fixtures with a check constraint that ensures only one target type is set.
- Tips and coupon items snapshot the odds selection fields needed by the product while optionally linking back to `odds.prematch_odds_current`.
- Featured fixtures can be manually curated or linked to an analytics signal.
- Notifications are separated from delivery attempts so in-app, push, and email delivery can be tracked independently.
- Contact messages are stored in PostgreSQL so the new app can keep a support inbox even when email delivery fails.
